## PRODUCT REQUIREMENTS DOCUMENT (PRD)

### 1. EXECUTIVE SUMMARY

The fundamental purpose of the Sensai Stack is to solve **The Market Paradox**: modern high-performance scaling environments invariably encounter a systemic ceiling where brittle operational infrastructure fractures under load, causing executive decision-making to collapse due to cognitive exhaustion and internal misalignment. The Sensai Stack deploys the Ontological Systems Architecture (OSA) to address this systemic entropy at its root.

This PRD outlines the structural and technical requirements for deploying an "Agentic OS" via Claude Code for Sunyata. To prevent scaling environments from fracturing under load and to eliminate executive cognitive exhaustion, this environment avoids scattered scripts and fragmented AI tools. Instead, it establishes a deterministic, low-entropy technical infrastructure utilizing terminal-based agentic ecosystems. Operating as an Agentic OS, it leverages strict multi-agent orchestration, platform-enforced isolation, and continuous state memory.

### 2. THE CORE OPERATIONAL AXIOM

* The firm operates on a literal architectural mirror, meaning the studio is itself the first build.
* Sunyata will run the consulting practice on the exact same five floors of infrastructure designed and deployed for clients.
* The architecture explicitly rejects the tendency to intellectually shrink, soften architectural terminology, or simplify conceptual depth to accommodate reactive, low-context spaces.

### 3. AGENTIC OS ARCHITECTURE: THE FIVE FLOORS

**Floor 0: Ground & Security Baseline**

* Identity and secrets are ringfenced using Proton Pass, SimpleLogin aliases, and hardware YubiKeys.
* Application-level credentials route exclusively through Doppler.
* This credential routing completely isolates environment secrets from code repositories and history tracking.
* All operational data vectors are defined within an immutable `DATA-MAP.md` manifest.
* This manifest explicitly tracks the encryption state, retention schedules, and mechanical deletion paths of all data.

**Floor 1: Speech & Instruction Layer**

* System operational behavior is governed via a layered `CLAUDE.md` architecture.
* This context hierarchy transitions from user invariants down to specific, isolated client engagement files.
* Under the Provenance Rule, every instruction must trace directly to a baseline foundation document, a legally binding client agreement, or a formally logged system failure.
* Following Claude Code configuration best practices, all instructions must utilize the "3C Rule" (Concise, Contextual, Constrained), while large execution requirements must be parsed via plan mode prior to autonomous tool use to prevent architectural drift.

**Floor 2: Sight, Calibration & Analytics**

* The Agentic OS integrates four core dashboards to track firm health: Pipeline, Delivery, Economics, and Content & Network.
* Custom tools, anonymizers, and prompt templates are verified programmatically using headless testing scripts.
* Changes to the system are gated.
* Any failure to clear regression tests completely halts the delivery pipeline until resolved.

**Floor 3: Witness & Memory Isolation**

* The firm enforces a strict four-tier data classification matrix to defend client privacy and prevent cross-contamination across active projects.
  * **Tier 0 Public**: Unrestricted access for essays, collateral suites, and public platform elements.
  * **Tier 1 Internal**: Internal system scope for financial ledgers, internal strategy logs, and the core corpus.
  * **Tier 2 Client-Coded**: Database tracking using strict codenames only for session notes, analytics matrices, and operational patterns.
  * **Tier 3 Client-Identified**: Completely isolated files containing legal contracts, invoices, and raw corpora mapped to dedicated, encrypted per-client local storage volumes.
* The master codename-to-identity mapping is stored exclusively inside hardware-backed, encrypted password managers.
* This enforces hardware-level isolation, ensuring the AI execution layer remains structurally blind to true client identities across shared environments.

**Floor 4: Trust & Platform-Enforced Automation**

* The practice avoids fragile graph runtimes or volatile multi-agent frameworks.
* The system opts for a deterministic, platform-enforced architecture driven by Claude Code and Claude Web via dedicated Projects.
* Claude Code functions as an integrated operations department of one.
* Automation is partitioned into three distinct subagents running inside isolated context windows, governed by strict tool allowlists rather than behavioral descriptions:
  * **Scribe**: Possesses limited tool rights to write directly to encrypted internal session repositories and update Linear checklists post-session.
  * **Quartermaster**: A low-overhead agent running read-only queries across internal metrics to assemble weekly dashboard deltas, flag charge-completion drops, and track API tool usage costs.
  * **Herald**: Manages the content assembly pipeline, pulling exclusively from anonymized content databases with zero structural permission to read raw client session folders.
* System guardrails are made absolute via deterministic shell hooks (PreToolUse and UserPromptSubmit), which intercept inputs programmatically.
* A bash blocklist guard utilizes `jq` to scan `.tool_input.content` and `.tool_input.command` to automatically prevent any identifying terms from leaking into model execution boundaries.

**Floor 5: The Watchtower (Native Observability)**

* The practice implements full OpenTelemetry (OTel) routing via a self-hosted, local OTel collector configured on an encrypted storage volume.
* This local watchtower natively ingests metrics, traces, and system logs from terminal execution layers.
* Session tracking is decoupled from true identity by exporting unique codenames as resource attributes at session initialization.
* This generates exact token cost attribution, telemetry auditing, and pipeline analytics without exposing client data to external cloud monitoring platforms.

### 4. ALGORITHMIC VALUATION & METRICS

To enforce pricing models that decouple manual execution hours from revenue, the Quartermaster subagent must track continuous state metrics aligned to the following values:

* **$\alpha$**: Programmatic savings derived directly from model context token optimization, prompt compression, and pipeline efficiency.
* **$\beta$**: Calculated value of total executive hours reclaimed by transitioning organizational workflows from live synchronous meetings to asynchronous data streams.
* **$\gamma$**: Premium assigned to isolating critical structural and legal choke points within the target jurisdiction.

### 5. ENVIRONMENTAL HYGIENE & DAILY PROTOCOL

A system is never more stable than its creator. To maintain the system without succumbing to the Lion-to-Raven Collapse Sequence, the operator environment must support strict biological alignment:

* All real-time client corporate communication channels (Slack, Microsoft Teams) maintain an absolute air gap.
* Strategic and technical handoffs occur solely via offline, asynchronous markdown document pipelines.
* Operational resilience treats the physiological and psychological substrate as a managed hardware-software stack.
* A strict 45-minute digital blackout post-waking protects attention from early dopamine hijacking.
* A protocol of targetless evening sitting is utilized to clear working memory, empty cognitive RAM, and completely decouple identity from technical production.
